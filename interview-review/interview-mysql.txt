索引: 普通索引 唯一索引 全文索引 联合索引
索引失效: 1.查询条件没有索引列 2.查询条件使用了or 3.like条件%在左 4.查询条件是string时没有使用''发生了类型转化,在索引上使用is null
联合索引 a,b,c  只有a,ab,abc可以命中. hash类型的索引只能在a,b,c命中
数据库三范式: 1,没有重复的列 2每一行必须被唯一区分 3.属性不依赖于其他非主属性
隔离级别: 读未提交,读提交,可重复读,序列化
myisam与innodb的区别: 
	1. innodb支持事务
	2. i支持行级锁, m支持表级锁.
	3. i支持外键,m不支持
	4. i不支持全文索引,m支持
varchar与char的区别:  char的存储是声明的固定长度. varchar会根据数据本身长度来存储

sql优化: 
	set profieling=true, explan sql : 查询sql语句的执行计划. 命中索引,是否全表查询,估计行数
	show full processcode ,查看是否存在阻塞的sql语句. 手动kill掉.
	sql语句:
		1. 合理使用索引,索引多查询速度快,但是插入速度会变慢.也会占用更多服务器资源
		2. 尽量命中索引,避免索引失效的情况.
		3. 使用union all 代替 union, union all 执行效率较高, union需要排重,排序
		4. 避免select * 写法
		5. join字段尽量建立索引
		6. 避免复杂sql
		7. 复杂sql可以拆分成若干简单sql

乐观锁与悲观锁:  	乐观锁使用版本号, 插入前比较版本号,不一致报错,否则修改时 版本号+1. 
		悲观锁 for update
